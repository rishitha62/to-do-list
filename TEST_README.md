## User Stories Covered

### 1. Add Tasks (`@DddTasks`)
- Successfully add a single task
- Add multiple tasks
- Cannot add empty task
- Cannot add whitespace-only task
- Add task using Enter key

### 2. View Added Tasks (`@ViewTasks`)
- View empty task list
- View task list with multiple tasks
- Verify task statistics display

### 3. Mark Tasks as Completed (`@CompleteTasks`)
- Mark a single task as completed
- Unmark a completed task
- Mark multiple tasks as completed
- Visual distinction with strike-through style

### 4. Delete Tasks (`@DeleteTasks`)
- Delete a single task
- Delete multiple tasks
- Delete a completed task
- Clear all completed tasks (batch delete)

### 5. User-Friendly Interface (`@UI`)
- Verify all UI elements are present
- Verify clear and organized layout

### 6. No Login or Registration
- Direct access to the application
- No authentication required

### 7. No Backend Requirement
- Client-side storage using localStorage
- All operations handled in the browser

### 8. Immediate Effect (No Page Reload)
- Tasks added without page reload
- Tasks marked as completed without page reload
- Tasks deleted without page reload

## Running Tests

### Run All Tests

```bash
mvn test
```

### Run Smoke Tests Only

Uncomment the `tags = "@Smoke"` line in `TestRunner.java` or run using:

```bash
mvn test -Dcucumber.filter.tags="@Smoke"
```

### Run Tests by Tag

```bash
# Run only Add Tasks tests
mvn test -Dcucumber.filter.tags="@DddTasks"

# Run only Delete Tasks tests
mvn test -Dcucumber.filter.tags="@DeleteTasks"

# Run only Complete Tasks tests
mvn test -Dcucumber.filter.tags="@CompleteTasks"
```

### Run Tests in Headless Mode

Tests are configured to run in headless mode by default. To run with a visible browser, comment out the `--headless` line in `TaskManagementSteps.java`.

## Configuration

### Changing the Application URL

By default, tests run against a local `index.html` file. To point to a different URL/path, update the `APP_URL` constant in `TaskManagementSteps.java`:

```java
private static final String APP_URL = "http://localhost:8080/index.html";
// OR
private static final String APP_URL = "file:///" + new File("path/to/index.html").getAbsolutePath();
```

### Changing the Browser

To use a different browser (e.g., Firefox), update the setup method in `TaskManagementSteps.java`:

```java
WebDriverManager.firefoxdriver().setup();
driver = new FirefoxDriver();
```

## Troubleshooting

### Issue: WebDriver not found

**Solution:** WebDriverManager automatically downloads the correct driver. Ensure you have an internet connection.

### Issue: Chrome browser not found

**Solution:** Install Google Chrome or update the test configuration to use a different browser.

### Issue: Tests fail with timeout errors

**Solution:** Increase the `WebDriverWait` timeout in `ToDoListPage.java`:

```java
this.wait = new WebDriverWait(driver, Duration.ofSeconds(20)); // Increase from 10 to 20 seconds
```

### Issue: Cannot find task elements

**Solution:** Ensure the index.html file is in the project root directory or update the `APP_URL` to point to the correct location.

## CI/CD Integration

These tests can be easily integrated into CI/CD pipelines:

### GitHub Actions Example

```yaml
name: BDD Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-java@v3
        with:
          java-version: '11'
      - name: Run Cucumber Tests
        run: mvn test
      - name: Upload Test Reports
        uses: actions/upload-artifact@v3
        with:
          name: cucumber-reports
          path: target/cucumber-reports/
```
